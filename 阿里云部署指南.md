# ServerScout 阿里云部署指南

## 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| Java JDK | 17+ | 运行 JAR |
| MySQL | 8.0+ | 数据库 |
| Nginx (可选) | 1.18+ | 反向代理/HTTPS |
| Nmap (可选) | 7.x | 端口扫描 |
| Nuclei (可选) | 3.x | 漏洞扫描 |

## 快速部署（前后端合并模式）

### 1. 本地打包

```bash
cd serverscout

# 构建前端
cd frontend
npm install
npm run build
cd ..

# 打包后端（含前端静态文件）
cd backend
mvn clean package -DskipTests
```

生成的 JAR 位于 `backend/target/server-scout-1.0.0.jar`（约89MB，已包含前端）。

### 2. 上传到阿里云 ECS

```bash
scp backend/target/server-scout-1.0.0.jar root@你的服务器IP:/opt/serverscout/
```

### 3. 服务器配置

#### 安装 Java 17

```bash
# CentOS/RHEL
yum install -y java-17-openjdk java-17-openjdk-devel

# Ubuntu/Debian
apt update && apt install -y openjdk-17-jdk
```

#### 创建数据库

```bash
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS serverscout CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

#### 安装中文字体（PDF 报告需要）

```bash
# CentOS
yum install -y google-noto-cjk-fonts

# Ubuntu
apt install -y fonts-noto-cjk
```

### 4. 启动服务

```bash
# 环境变量配置
export SERVER_PORT=8080
export DB_URL="jdbc:mysql://localhost:3306/serverscout?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai"
export DB_USERNAME="root"
export DB_PASSWORD="你的数据库密码"
export JWT_SECRET="生成一个随机字符串（至少32字符）"
export LOG_LEVEL="INFO"
export PDF_FONT_PATH="/usr/share/fonts/truetype/noto/NotoSansCJK-Regular.ttc"

# 启动
cd /opt/serverscout
java -Xms512m -Xmx1024m -jar server-scout-1.0.0.jar
```

### 5. 使用 systemd 管理（推荐）

创建 `/etc/systemd/system/serverscout.service`：

```ini
[Unit]
Description=ServerScout Platform
After=network.target mysql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/serverscout
ExecStart=/usr/bin/java -Xms512m -Xmx1024m -jar /opt/serverscout/server-scout-1.0.0.jar
Restart=on-failure
RestartSec=10

Environment="SERVER_PORT=8080"
Environment="DB_URL=jdbc:mysql://localhost:3306/serverscout?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai"
Environment="DB_USERNAME=root"
Environment="DB_PASSWORD=你的数据库密码"
Environment="JWT_SECRET=你的JWT密钥"
Environment="LOG_LEVEL=INFO"

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable serverscout
systemctl start serverscout
systemctl status serverscout
```

### 6. 配置 Nginx 反向代理（可选但推荐）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    client_max_body_size 10m;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 7. 配置 HTTPS（推荐使用 certbot）

```bash
# 安装 certbot
yum install -y certbot python3-certbot-nginx

# 获取证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

## 阿里云安全组配置

在阿里云 ECS 控制台 → 安全组 → 添加规则：

| 端口 | 协议 | 授权对象 | 说明 |
|------|------|----------|------|
| 80 | TCP | 0.0.0.0/0 | HTTP |
| 443 | TCP | 0.0.0.0/0 | HTTPS |
| 8080 | TCP | 0.0.0.0/0 | 后端直连（可选，建议仅内网） |
| 3306 | TCP | 127.0.0.1 | MySQL（仅本地访问） |

## 验证部署

```bash
# 检查服务状态
systemctl status serverscout

# 查看日志
journalctl -u serverscout -f

# 测试 API
curl http://localhost:8080/api/auth/captcha

# 测试前端
curl http://localhost:8080/
```

浏览器访问 `http://你的服务器IP:8080` 即可使用。

## 故障排查

1. **端口被占用**：修改 `SERVER_PORT` 环境变量
2. **MySQL 连接失败**：检查数据库是否创建、密码是否正确、是否允许远程连接
3. **PDF 中文乱码**：确认中文字体已安装，检查 `PDF_FONT_PATH` 配置
4. **Nmap/Nuclei 扫描不可用**：在 ECS 上安装对应工具，在系统设置中配置路径
