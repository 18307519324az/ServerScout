## 1. 详细 API 设计

### 1.1 通用规范

**基础路径**: `/api/v1`

**统一响应格式**:
```json
{
    "code": 200,
    "message": "success",
    "data": {},
    "timestamp": "2026-05-19T10:00:00Z"
}
```

**错误响应格式**:
```json
{
    "code": 4000,
    "message": "参数校验失败",
    "data": {
        "field": "target_range",
        "error": "IP 格式不正确"
    },
    "timestamp": "2026-05-19T10:00:00Z"
}
```

**错误码定义**:

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 4000 | 参数校验失败 |
| 4001 | 资源不存在 |
| 4002 | 状态冲突（如任务已在运行） |
| 4003 | 扫描目标格式错误 |
| 5000 | 服务器内部错误 |
| 5001 | 扫描引擎调用失败 |
| 5002 | 报告生成失败 |
| 5003 | 数据库操作失败 |

**分页请求参数**: `?page=0&size=20&sort=createdAt,desc`

**分页响应格式**:
```json
{
    "code": 200,
    "data": {
        "content": [],
        "page": 0,
        "size": 20,
        "totalElements": 156,
        "totalPages": 8,
        "first": true,
        "last": false
    }
}
```

### 1.3 扫描任务 API

#### POST /api/v1/scan-tasks — 创建扫描任务

**请求**:
```json
{
    "name": "内网资产巡检-20260519",
    "targetRange": "192.168.1.0/24",
    "scanType": "quick",
    "portRange": "1-1000",
    "enableFingerprint": true,
    "enableVulnScan": false,
    "config": {
        "timeout": 300,
        "concurrency": 50,
        "rateLimit": 1000
    }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 任务名称 |
| targetRange | String | 是 | 目标 IP/CIDR/域名 |
| scanType | String | 否 | quick/full/custom，默认 quick |
| portRange | String | 否 | 端口范围，默认 1-1000 |
| enableFingerprint | Boolean | 否 | 是否启用指纹识别 |
| enableVulnScan | Boolean | 否 | 是否启用漏洞扫描 |

**响应**:
```json
{
    "code": 200,
    "data": {
        "id": 1,
        "name": "内网资产巡检-20260519",
        "status": "pending",
        "createdAt": "2026-05-19T10:00:00Z"

}
```

### 2.4 关键交互流程

**创建扫描任务流程**:
```
用户点击"新建扫描"
    → 弹出 CreateTaskDialog
    → 输入目标 IP/CIDR（格式实时校验）
    → 选择扫描类型和选项
    → 展开高级配置（可选）
    → 点击"开始扫描"
    → POST /api/v1/scan-tasks
    → 关闭对话框
    → React Query 刷新任务列表
    → 自动跳转至任务详情页
    → 页面通过 SSE 实时接收进度
    → 进度条动画更新
    → 扫描完成 → 提示用户
```

**资产详情浏览流程**:
```
用户从资产列表点击某资产
    → 跳转 /assets/:id
    → GET /api/v1/assets/{id}
    → 左侧显示资产基本信息 + 标签
    → 右侧显示端口列表（可展开）
    → 展开端口 → 显示服务指纹 + Web 指纹
    → 如果有关联漏洞 → 显示在端口下方
    → 点击"查看拓扑"按钮
    → 弹出拓扑图弹窗
    → D3.js 渲染力导向图
    → 悬停节点 → 显示资产摘要
    → 点击节点 → 跳转至该资产详情
```

## 3. 数据库详细设计

### 3.1 DDL 语句

```sql
-- ============================================
-- ServerScout 数据库 DDL
-- PostgreSQL 16
-- ============================================

-- 扫描任务表
CREATE TABLE scan_task (
    id              BIGSERIAL PRIMARY KEY,
    name            VARCHAR(128)    NOT NULL,
    target_range    VARCHAR(512)    NOT NULL,
    scan_type       VARCHAR(32)     NOT NULL DEFAULT 'quick',
    port_range      VARCHAR(64)     DEFAULT '1-1000',
    enable_fingerprint BOOLEAN      DEFAULT TRUE,
    enable_vuln_scan   BOOLEAN      DEFAULT FALSE,
    status          VARCHAR(16)     NOT NULL DEFAULT 'pending',
    progress        INTEGER         NOT NULL DEFAULT 0,
    total_assets    INTEGER         NOT NULL DEFAULT 0,
    total_ports     INTEGER         NOT NULL DEFAULT 0,
    config_json     TEXT,
    error_message   TEXT,
    started_at      TIMESTAMP WITH TIME ZONE,
    completed_at    TIMESTAMP WITH TIME ZONE,
    created_by      VARCHAR(64),
    created_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

-- 资产表
CREATE TABLE asset (
    id                  BIGSERIAL PRIMARY KEY,
    task_id             BIGINT          NOT NULL REFERENCES scan_task(id) ON DELETE CASCADE,
    ip_address          VARCHAR(45)     NOT NULL,
    hostname            VARCHAR(256),
    os_fingerprint      VARCHAR(128),
    os_version          VARCHAR(64),
    status              VARCHAR(16)     NOT NULL DEFAULT 'unknown',
    open_port_count     INTEGER         NOT NULL DEFAULT 0,
    critical_vuln_count INTEGER         NOT NULL DEFAULT 0,
    tags                JSONB           DEFAULT '[]',
    discovered_at       TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_asset_task_ip UNIQUE (task_id, ip_address)
);

-- 端口表
CREATE TABLE port (
    id              BIGSERIAL PRIMARY KEY,
    asset_id        BIGINT          NOT NULL REFERENCES asset(id) ON DELETE CASCADE,
    port_number     INTEGER         NOT NULL,
    protocol        VARCHAR(8)      NOT NULL DEFAULT 'tcp',
    service_name    VARCHAR(64),
    service_version VARCHAR(128),
    service_product VARCHAR(128),
    state           VARCHAR(16)     NOT NULL DEFAULT 'open',
    banner          TEXT,
    is_web_service  BOOLEAN         NOT NULL DEFAULT FALSE,
    first_seen_at   TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    CONSTRAINT uq_asset_port_protocol UNIQUE (asset_id, port_number, protocol)
);

-- Web 指纹表
CREATE TABLE web_fingerprint (
    id                  BIGSERIAL PRIMARY KEY,
```

## 5. 关键技术难点与方案

### 5.1 Nmap CLI 异步调用与结果解析

**难点**: Nmap 扫描是耗时的 IO 操作，需要异步执行且不阻塞主线程；解析 XML 结果需要高效可靠。

**方案**:

```java
@Service
public class NmapScannerImpl implements ScannerStrategy {

    @Async("scanExecutor")  // 使用独立线程池
    public CompletableFuture<ScanResult> execute(ScanTask task) {
        // 1. 构建 Nmap 命令
        List<String> command = buildNmapCommand(task);

        // 2. 使用 ProcessBuilder 执行
        ProcessBuilder pb = new ProcessBuilder(command);
        pb.redirectErrorStream(true);
        Process process = pb.start();

        // 3. 异步读取输出（防止缓冲区阻塞）
        StringBuilder output = new StringBuilder();
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(process.getInputStream()))) {
            String line;
            while ((line = reader.readLine()) != null) {
                output.append(line).append("\n");
                // 可选：解析进度行，更新任务进度
            }
        }

        // 4. 等待完成
        int exitCode = process.waitFor();

        // 5. 解析 XML 输出
        if (exitCode == 0) {
            return parseNmapXmlOutput(output.toString());
        } else {
            throw new ScanException("Nmap 执行失败，退出码: " + exitCode);
        }
    }

    private List<String> buildNmapCommand(ScanTask task) {
        List<String> cmd = new ArrayList<>(Arrays.asList("nmap"));
        cmd.add("-oX");         // XML 输出
        cmd.add("-");           // 输出到 stdout
        cmd.add("--stats-every"); cmd.add("5s");  // 每 5 秒输出进度

        switch (task.getScanType()) {
            case "quick":
                cmd.add("-sn");  // Ping 扫描
                break;
            case "full":
                cmd.add("-sS");  // SYN 扫描
                cmd.add("-sV");  // 版本检测
                cmd.add("-O");   // OS 检测
                break;
        }

        cmd.add(task.getTargetRange());
        return cmd;
    }
}
```

### 5.2 XML 结果解析

**难点**: Nmap XML 输出结构复杂，包含 hosts、ports、os 等多层嵌套。

**方案**: 使用 Jackson XmlMapper 或 DOM 解析器

```java
// 使用 DOM 解析 Nmap XML
public ScanResult parseNmapXmlOutput(String xml) {
    DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
    DocumentBuilder builder = factory.newDocumentBuilder();
    Document doc = builder.parse(new InputSource(new StringReader(xml)));

    ScanResult result = new ScanResult();
    NodeList hostNodes = doc.getElementsByTagName("host");

    for (int i = 0; i < hostNodes.getLength(); i++) {
        Element hostEl = (Element) hostNodes.item(i);
        AssetResult asset = new AssetResult();

        // 提取 IP
        Element address = (Element) hostEl.getElementsByTagName("address").item(0);
        asset.setIpAddress(address.getAttribute("addr"));

        // 提取主机名
        Element hostname = (Element) hostEl.getElementsByTagName("hostname").item(0);
        if (hostname != null) {
            asset.setHostname(hostname.getAttribute("name"));
        }

        // 提取 OS 指纹
        Element os = (Element) hostEl.getElementsByTagName("os").item(0);
        if (os != null) {
            Element osMatch = (Element) os.getElementsByTagName("osmatch").item(0);
            if (osMatch != null) {
                asset.setOsFingerprint(osMatch.getAttribute("name"));
            }
        }

        // 提取端口
        Element ports = (Element) hostEl.getElementsByTagName("ports").item(0);
        if (ports != null) {
            NodeList portNodes = ports.getElementsByTagName("port");
            for (int j = 0; j < portNodes.getLength(); j++) {
                Element portEl = (Element) portNodes.item(j);
                PortResult port = new PortResult();
                port.setPortNumber(Integer.parseInt(portEl.getAttribute("portid")));
                port.setProtocol(portEl.getAttribute("protocol"));

                Element state = (Element) portEl.getElementsByTagName("state").item(0);
                port.setState(state.getAttribute("state"));

                Element service = (Element) portEl.getElementsByTagName("service").item(0);
                if (service != null) {
                    port.setServiceName(service.getAttribute("name"));
                    port.setServiceVersion(service.getAttribute("version"));
                    port.setServiceProduct(service.getAttribute("product"));
                }
                asset.getPorts().add(port);
            }
        }
        result.getAssets().add(asset);
    }
    return result;
}
```

### 5.3 SSE 实时进度推送

**难点**: 前端需要实时看到扫描进度，但轮询效率低。

**方案**: 使用 Spring WebFlux 的 SseEmitter（无需额外依赖）

```java
@RestController
@RequestMapping("/api/v1/scan-tasks")
public class ScanTaskController {

    // 存储每个任务的 SSE Emitter
    private final ConcurrentHashMap<Long, List<SseEmitter>> taskEmitters = new ConcurrentHashMap<>();

    @GetMapping("/{id}/progress")
    public SseEmitter streamProgress(@PathVariable Long id) {
        SseEmitter emitter = new SseEmitter(0L);  // 不超时

        taskEmitters.computeIfAbsent(id, k -> new CopyOnWriteArrayList<>())
                    .add(emitter);

        emitter.onCompletion(() -> removeEmitter(id, emitter));
        emitter.onTimeout(() -> removeEmitter(id, emitter));

        return emitter;
    }

    // 扫描执行器中调用此方法推送进度
    public void sendProgress(Long taskId, int progress, String currentTarget, int assetsFound) {
        List<SseEmitter> emitters = taskEmitters.get(taskId);
        if (emitters == null) return;

        Map<String, Object> data = Map.of(
            "progress", progress,
            "currentTarget", currentTarget,
            "assetsFound", assetsFound
        );

        for (SseEmitter emitter : emitters) {
            try {
                emitter.send(SseEmitter.event()
                    .name("progress")
                    .data(data));
            } catch (IOException e) {
                emitter.completeWithError(e);
                emitters.remove(emitter);
            }
        }
    }
}
```

**前端消费 SSE**:

```typescript
// 前端 React Hook
function useScanProgress(taskId: number) {
    const [progress, setProgress] = useState(0);
    const [currentTarget, setCurrentTarget] = useState('');

    useEffect(() => {
        if (!taskId) return;

        const eventSource = new EventSource(
            `/api/v1/scan-tasks/${taskId}/progress`
        );

        eventSource.addEventListener('progress', (event) => {
            const data = JSON.parse(event.data);
            setProgress(data.progress);
            setCurrentTarget(data.currentTarget);
        });

        eventSource.addEventListener('completed', () => {
            eventSource.close();
            // 触发数据刷新
            queryClient.invalidateQueries(['scan-task', taskId]);
        });

        eventSource.addEventListener('error', () => {
            eventSource.close();
        });

        return () => eventSource.close();
    }, [taskId]);

    return { progress, currentTarget };
}
```

### 5.4 批量数据写入优化

**难点**: 扫描 254 个 IP 的 /24 网段可能产生数千条端口记录，逐条 INSERT 性能差。

**方案**: JPA 批量写入 + 手动批次提交

```yaml

spring:
  jpa:
    properties:
      hibernate:
        jdbc.batch_size: 50

```

```java
// 批量写入服务
@Service
@Transactional
public class AssetPersistenceService {

    @Autowired
    private AssetRepository assetRepository;

    public void batchSaveAssets(List<Asset> assets, int batchSize) {
        EntityManager em = entityManagerFactory.createEntityManager();
        EntityTransaction tx = em.getTransaction();
        try {
            tx.begin();
            for (int i = 0; i < assets.size(); i++) {
                em.persist(assets.get(i));
                if (i > 0 && i % batchSize == 0) {
                    em.flush();
                    em.clear();
                }
            }
            tx.commit();
        } catch (Exception e) {
            tx.rollback();
            throw e;
        } finally {
            em.close();
        }
    }
}
```
