# 测试与验收

## 后端启动验收

- 执行 `mvn spring-boot:run`
- 访问 `http://localhost:8080/docs`
- 确认接口文档页面可以正常打开

## 前端启动验收

- 执行 `npm run dev`
- 访问 `http://localhost:5173`
- 确认首页和登录页面可正常渲染

## 数据库连接验收

- 确认后端启动时无数据库连接报错
- 确认相关表结构已创建或迁移成功

## 登录流程验收

- 使用可用账号尝试登录
- 检查 Token 是否返回
- 检查受保护页面是否可以访问

## 资产管理验收

- 检查资产列表是否可正常加载
- 检查资产详情中端口、证书、漏洞、指纹等信息是否可查看

## 扫描任务验收

- 创建扫描任务
- 检查任务状态变化
- 检查 SSE 进度流是否可接收

## 报告导出验收

- 触发 PDF 导出
- 触发 Excel 导出
- 检查导出文件是否可下载、内容是否完整

## AI 风险简报验收

- 输入示例证据
- 检查是否能生成风险摘要、优先级与建议
- 若未配置模型接口，检查本地兜底逻辑是否可运行

## Docker Compose 验收

- 执行 `docker-compose up -d --build`
- 检查 MySQL 与应用容器是否正常启动
- 检查容器日志是否存在异常

## 安全合规检查

- 确认仅用于授权资产场景
- 确认未提交真实 Token、密钥或数据库密码
- 确认 JWT 密钥、数据库密码使用环境变量注入

## 文档链接检查

- 检查 README 中的所有文档导航是否可点击
- 检查 `docs` 中文件名称与链接是否一致

## 敏感信息检查

- 检查仓库中是否存在真实密钥、Token 或明文密码
- 检查 `.env` 是否未被提交
- 检查示例配置是否仅保留占位值

## 验收结论建议

在正式展示前，至少完成一次后端启动、前端启动、数据库连接、扫描任务创建、报告导出与 AI 简报的完整联调检查。
