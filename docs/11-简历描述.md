# 简历描述

## 1. 简历项目名称

**ServerScout 服务器资产攻击面管理与风险分析平台**

## 2. 项目描述

基于 Spring Boot 3 和 React 18 构建的服务器资产攻击面管理与风险分析平台。集成 Nmap 和 Nuclei 扫描引擎，实现资产发现、端口与服务识别、漏洞检测、CVE 关联、5 因子风险评分量化、攻击面拓扑可视化、PDF/Excel 报告导出和 AI 风险简报生成。项目采用策略模式解耦扫描引擎，9 阶段 Pipeline 状态机追踪异步扫描进度，Demo Mode 支持零环境依赖的完整功能演示。

| 类别 | 技术 |

## 4. 核心职责

- 独立完成从需求分析、系统设计、前后端开发到 Docker 部署的全流程
- 设计并实现扫描策略模式（ScannerStrategy 接口），解耦 Demo/Nmap/Nuclei 三种扫描策略
- 设计并实现 9 阶段 Pipeline 状态机，支持异步扫描进度实时追踪
- 设计并实现 5 因子加权风险评分模型，含事务安全的 Recalculate 机制
- 设计并实现 PDF/Excel 双格式报告导出，解决中文渲染问题
- 实现 JWT 认证、RBAC 权限控制、统一异常处理、操作日志审计
- 编写 96 个单元/集成测试，覆盖率覆盖服务层、控制器层和异常处理
- 配置 GitHub Actions CI（Maven 测试 + 前端构建 + Docker Compose 验证）

## 5. 技术亮点

1. **策略模式解耦扫描引擎**：ScannerStrategy 接口定义扫描契约，Demo/Nmap/Nuclei 三种策略可插拔替换。Demo Mode 在零工具环境下跑通完整业务闭环，设计时同步验证了接口抽象的合理性
2. **9 阶段 Pipeline 状态机**：每个阶段支持 PENDING/RUNNING/SUCCESS/FAILED/SKIPPED 五种状态，异步执行时前端轮询实时展示进度百分比；失败阶段不阻塞已保存数据
3. **5 因子风险评分模型**：暴露面×25% + 漏洞严重性×30% + 服务风险×15% + 可利用性×15% + 业务重要性×15% - 修复扣减，0-100 分量化；无漏洞资产也有基础分；Recalculate 使用 @Transactional 保证删除+重算的原子性
4. **全栈工程闭环**：从扫描引擎集成（Nmap/Nuclei shell 调用 → 结构化解析 → CVE 关联 → 风险评级）到前端可视化（仪表盘 + 拓扑图 + 阶段进度）到 DevOps（Docker 多阶段构建 + CI），形成了完整的业务闭环
5. **可演示性优先**：Demo Mode 默认开启，内置模拟数据生成器、预置端口池和 CVE 库，不依赖任何外部工具或真实目标，现场开机即用

## 6. 面试讲解版本（5 分钟）

ServerScout 是一个面向服务器资产攻击面管理的全栈项目。后端使用 Spring Boot 3 实现扫描任务编排、工具集成、风险评分、报告导出；前端使用 React 18 实现资产管理、拓扑可视化、报告中心和 AI 简报。

项目的核心在于三个特色设计：Demo Mode 让项目在零依赖环境下完整演示扫描闭环；9 阶段 Pipeline 状态机实现扫描进度的细粒度追踪；5 因子加权评分模型将安全风险量化为 0-100 的可解释分数。

技术上使用了策略模式解耦三种扫描策略、@Transactional 保证评分重算的事务安全、Apache POI 和 iText 7 实现中文报告导出。项目包含 96 个测试用例，通过 GitHub Actions CI 执行 Maven 测试、前端构建和 Docker Compose 静态校验。

## 8. 5 条项目经历 Bullet

- 基于 Spring Boot 3 和 React 18 独立开发服务器资产攻击面管理平台，集成 Nmap 端口探测与 Nuclei 漏洞检测引擎，覆盖资产发现→漏洞扫描→风险评分→报告导出全流程
- 设计策略模式（ScannerStrategy 接口）解耦扫描引擎，实现 Demo/Nmap/Nuclei 三种策略可插拔替换；Demo Mode 在零环境依赖下跑通完整业务闭环

## 9. 可量化描述

- 后端 96 个测试用例，0 failures，测试覆盖业务服务层、控制器层、扫描策略层
