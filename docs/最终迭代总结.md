### 1.2 角色权限控制

JWT Token 中加入 `role` 声明，实现前后端双重权限拦截：

- **后端**：`SecurityConfig` 提取 `ROLE_ADMIN`/`ROLE_USER` 权限；`UserController` 全部管理端点加 `@PreAuthorize("hasRole('ADMIN')")`；`/api/v1/users/**` 仅 ADMIN 可访问
- **前端**：`localStorage` 存储 role；SettingsPage 判断 `isAdmin` —— ADMIN 显示完整用户管理，USER 显示"权限不足"

## 4. 遇到的困难与教训

### 4.1 困难 1：Java 8 vs Java 17 版本冲突

**现象**：`mvn spring-boot:run` 报 ClassNotFoundException，但 `mvn compile` 成功。

**根因**：系统 PATH 中 `java` 指向 JDK 8，Spring Boot 3.3.5 需要 JDK 17。

**解决**：必须使用完整路径 `"C:/Program Files/Java/jdk-17/bin/java" -jar target/server-scout-1.0.0.jar` 显式指定 JDK 17。`mvn` 也需设置 `JAVA_HOME`。

**教训**：在 Windows 多 JDK 环境下，包装启动脚本是必要的。

### 4.2 困难 2：CORS 端口不匹配导致 403

**现象**：前端登录返回 403，但 curl 测试正常。

**根因**：CORS 配置只允许 `localhost:5173`，实际 Vite 因端口被占用启动在 `5174`。

**解决**：`WebConfig` 添加 `5174` 到 allowedOrigins。

### 4.3 困难 3：Maven 编译与文件锁

**现象**：修改 Java 文件后 `mvn package` 报错 `Unable to rename JAR to JAR.original`。

**根因**：运行中的 JAR 被 `target/` 目录锁定。

**解决**：先 `taskkill -F -IM java.exe`，再 `mvn clean package`。

### 4.4 困难 4：Python × 符号与 bash 编码

**现象**：Bash 脚本中 Python 解析验证码 `×` 符号失败，答案为空。

**根因**：`×` 是 U+00D7（乘号），Python 字符串比较需精确匹配。此外 bash 中 `2>&1` 导致 stderr 混入变量。

**解决**：改用 bash 原生 `case "$OP" in "×")` 解析算术题。

### 4.5 困难 5：E2E 测试中 localStorage 残留

**现象**：浏览器中登录 admin 后 `localStorage.getItem('role')` 仍返回 "USER"。

**根因**：前次测试 login 的 `fill()` selector 匹配到了验证码框而非用户名框（`input[type="text"]` 同时匹配两者）。

**解决**：用 `placeholder` 属性精确定位每个输入框。

### 4.6 本轮教训总结

1. **JWT 必须包含角色信息**：否则每次请求都要查数据库
2. **前后端双重权限校验**：前端隐藏UI提升体验，后端 `@PreAuthorize` 保证安全
3. **验证码用 ConcurrentHashMap + 定时清理**：简单有效，不需要 Redis
4. **Toast 通知显著提升 UX**：替代 `window.alert()` 和无声失败
5. **骨架屏 vs 加载中文字**：前者大福提升感知性能
6. **E2E 测试注意 input selector 精确性**：`input[type="text"]` 在大表单中不可靠，用 placeholder 更准
7. **报告导出必须指定有效 taskId**：测试前先查数据库确认存在的 ID
8. **Windows 环境 JDK 版本管理需脚本化**：避免 `mvn` 和 `java` 使用不同版本
