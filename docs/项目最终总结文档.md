# ServerScout 毕业设计项目 — 最终总结文档

> **项目名称**：ServerScout — 服务器资产攻击面可视化分析平台  
> **完成时间**：2026年5月  
> **版本**：v1.0.0

---

## 一、项目概述

ServerScout 是一套面向安全从业者的资产攻击面管理平台，支持资产发现、端口扫描、漏洞检测、Web指纹识别、蜜罐识别、网络拓扑可视化、外部威胁情报查询、报告导出等完整安全评估流程。

### 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | React + TypeScript | 18.x |
| 构建工具 | Vite | 5.x |
| UI 框架 | Tailwind CSS + Ant Design | - |
| 图表/可视化 | ECharts / G6 / D3.js | - |
| 后端框架 | Spring Boot | 3.3.5 |
| JDK | Java | 17 |
| 数据库 | MySQL | 8.0 |
| ORM | Hibernate / Spring Data JPA | 6.5 |
| 安全 | Spring Security + JWT + RSA | - |
| 扫描引擎 | Nmap + Nuclei | - |
| 报告 | iText 7 (PDF) + Apache POI (Excel) | - |
| 部署 | 宝塔面板 + 阿里云 ECS | - |

## 五、遇到的困难与解决方案

### 1. MySQL 5.7 兼容性问题

**问题**：阿里云服务器 MySQL 版本为 5.7.44，Hibernate 6.5 官方仅支持 8.0+，启动时有 Dialect 警告，且某些字段类型转换行为不一致。

**解决**：
- 手动编写完整的 SQL 初始化脚本，确保表结构与 JPA 实体严格匹配
- 将 `ddl-auto` 设为 `update` 作为容错，同时提供 SQL 导入方案
- 关注 `created_at` 默认值、`json` 类型映射等细节

### 2. 演示数据初始化导致应用崩溃

**问题**：`DemoDataInitializer` 在插入 `web_fingerprint` 时失败，因为 SQL 脚本创建的表缺少 `created_at` 默认值，且 `@Transactional` 导致事务回滚，即使捕获异常也抛出 `UnexpectedRollbackException` 导致应用退出。

**解决**：
- 去掉 `run()` 方法的 `@Transactional` 注解，让每步操作独立事务
- 包裹 try-catch，失败时输出 WARN 而非崩溃
- 教训：`@Transactional` 方法内 catch 异常无法阻止回滚，事务边界要合理设计

### 3. 前端静态资源与后端同端口部署

**问题**：开发时前后端分离（Vite :5173 + Spring Boot :8080），部署时需合并为单端口。

**解决**：
- Maven 配置 `resources` 将 `frontend/dist/` 打包进 `BOOT-INF/classes/static/`
- WebConfig 实现 SPA fallback：任何非 API 路径返回 `index.html`
- 解决了打包后前端路由（如 `/dashboard`）404 的问题

### 4. 宝塔面板 Java 项目部署

**问题**：宝塔的 Java 项目管理器对 Spring Boot 3 + JDK 17 的支持有限，环境变量占位符解析与本地不同。

**解决**：
- 将 `application.yml` 中的 `${VAR:default}` 占位符改为硬编码值
- JDK 必须选 17，宝塔默认可能选 8
- 监听在 `0.0.0.0` 确保外网可访问

### 5. 移动端响应式适配

**问题**：部分页面使用固定宽度（`w-96`、`w-[500px]`），在移动端溢出。

**解决**：
- 登录卡片改为 `w-full max-w-96`
- 对话框改为 `max-w-[500px] mx-4`
- 表格添加 `overflow-x-auto` + `min-w-[xxx]`
- Header 使用 `flex-col sm:flex-row` 自适应
- 资产列表增加移动端卡片视图（`md:hidden` / `hidden md:block`）

### 6. 多租户数据隔离

**问题**：演示数据全部归属于 admin 账号，新增普通用户后资产列表为空（预期行为），但需向用户说明数据归属逻辑。

**设计**：
- 资产通过 `scan_task.created_by` 关联到用户
- Admin 角色看到全部资产
- 普通用户仅看到自己创建扫描任务下的资产
- `AssetService.listAssets()` 通过 `searchByCreatedBy()` JOIN 查询实现过滤

### 7. PDF 中文字体

**问题**：iText 7 生成 PDF 报告时中文显示为空白方块。

**解决**：
- Windows 环境使用系统字体 `C:/Windows/Fonts/msyh.ttc`
- Linux 环境安装 `google-noto-cjk-fonts`，路径 `/usr/share/fonts/google-noto-cjk/NotoSansCJK-Regular.ttc`
- 通过 `app.pdf.font-path` 配置项灵活切换

### 8. RSA 加密与前端对接

**问题**：登录密码需加密传输，前端 jsencrypt 与后端 RSA 密钥对需匹配。

**解决**：
- 后端 `RsaKeyProvider` 启动时生成 2048-bit 密钥对
- 前端通过 `/api/auth/public-key` 获取公钥后加密
- 后端用私钥解密 → BCrypt 验证

## 六、部署配置要点

### 阿里云 + 宝塔面板部署清单

1. **JDK 17**：宝塔软件商店安装 Java 项目管理器
2. **MySQL**：创建 `serverscout` 数据库 + 导入 SQL 初始化脚本
3. **JAR 部署**：上传到 `/www/wwwroot/ServerScout/`，端口 8080
4. **安全组**：阿里云 ECS + 宝塔安全 双重放行 8080
5. **扫描工具**：`yum install -y nmap` + `go install nuclei`
6. **中文字体**：`yum install -y google-noto-cjk-fonts`（PDF 报告必需）
7. **Nginx 反代**（可选）：域名 80/443 → `http://127.0.0.1:8080`

### 环境变量说明

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `SERVER_PORT` | 8080 | 服务端口 |
| `LOG_LEVEL` | INFO | 日志级别 |
| `JWT_SECRET` | 内置默认 | JWT 签名密钥 |
| `NMAP_PATH` | nmap | Nmap 执行路径 |
| `NUCLEI_PATH` | nuclei | Nuclei 执行路径 |
| `PDF_FONT_PATH` | Linux 字体路径 | PDF 中文字体 |
