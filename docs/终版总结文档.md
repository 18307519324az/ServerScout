## 一、项目概述

ServerScout 是一个轻量级服务器资产攻击面可视化分析平台，面向中小型企业安全运维人员，提供从资产发现、指纹识别、漏洞关联到报告导出的全流程攻击面管理能力。

**技术栈**: React 18 + TypeScript + Vite + Tailwind CSS + Spring Boot 3.3.5 + Java 17 + MySQL 8.0

## 六、遇到的困难与教训

### 6.1 技术难点

**1. Spring Boot CommandLineRunner 执行顺序问题**
- 问题：DemoDataInitializer(@Order(2)) 依赖 CveDataInitializer 先执行，但 CveDataInitializer 未指定 @Order 默认为最低优先级
- 教训：`@Order` 数值越小越先执行；多 Runner 依赖时务必显式声明所有优先级

**2. JPA LazyInitializationException**
- 问题：CommandLineRunner 的 run() 方法不在事务上下文中，stream 遍历 LAZY 关联实体时 Session 已关闭
- 教训：涉及 LAZY 实体读取的 Runner 方法必须加 @Transactional

**3. React useEffect 无限渲染循环**
- 问题：useEffect 依赖 useQuery 返回的对象引用（每次 render 都不同），导致 setState → re-render → useEffect 的无限循环
- 教训：useEffect 依赖应使用 primitive 值（如 `data?.fieldName`），配合 useRef 防重

**4. Windows `where` 命令输出本地化**
- 问题：中英文 Windows 的 `where` 命令在找不到工具时返回不同的错误消息（中文"信息: 用提供的模式无法找到文件"），导致误判为有效路径
- 教训：跨平台命令行工具必须处理多语言输出，用正则匹配多种语言的关键错误信息

### 6.2 过程经验

- **E2E 浏览器测试价值巨大**：暗色模式下 5 个页面的文字不可见问题，仅靠代码 review 无法发现，必须实际渲染验证
- **级联删除需要仔细设计**：JPA 的 `cascade = CascadeType.REMOVE` 在复杂关联中容易遗漏约束，手动逐层删除更可控
- **配置外置是答辩加分项**：将密钥和路径移到环境变量虽然代码量不大，但体现了"生产环境意识"
- **演示数据至关重要**：12 个目标 + 11 条漏洞的数据集让平台在答辩时看起来"像真的在用"
