# 本地启动

## 环境准备

- Java 17 或更高版本
- Maven 3.9 或更高版本
- Node.js 18 或更高版本
- MySQL 8.0
- 可选：Redis
- 可选：Nmap、Nuclei

## Java / Maven 要求

- 后端基于 Spring Boot 3 和 Java 17
- 启动前请确认 `java -version` 与 `mvn -version` 可正常执行

## Node.js / npm 要求

- 前端基于 React 18 和 Vite
- 启动前请确认 `node -v` 与 `npm -v` 可正常执行

## MySQL 初始化

创建数据库后执行初始化脚本：

```bash
mysql -u root -p < serverscout-init.sql
```

如果使用自定义数据库地址，请同步调整后端配置或环境变量。

## 启动后端

```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

默认访问地址：

- `http://localhost:8080`
- Swagger / OpenAPI：`http://localhost:8080/docs`

## 启动前端

```bash
cd frontend
npm install
npm run dev
```

默认访问地址：

- `http://localhost:5173`

## 使用 Docker Compose 启动

```bash
docker-compose up -d --build
```

首次使用前建议复制 `.env.example` 并修改数据库密码、JWT 密钥等配置。

## Nmap 安装说明

- Windows：安装 Nmap 并确保可执行文件加入系统 PATH
- Linux：通过包管理器安装，例如 `apt install nmap`
- 如未加入 PATH，可通过 `NMAP_PATH` 显式指定可执行路径

## Nuclei 安装说明

- 从官方发布渠道安装 Nuclei
- 确认 `nuclei -version` 可执行
- 如未加入 PATH，可通过 `NUCLEI_PATH` 显式指定可执行路径

## 常见启动问题

- 前端能启动但接口报错：检查后端服务是否已启动
- 后端连接数据库失败：检查 MySQL 地址、账号、密码与数据库初始化情况
- 扫描任务无法执行：检查 Nmap、Nuclei 是否安装且路径正确
- PDF 中文乱码：检查系统字体是否存在，并正确设置 `PDF_FONT_PATH`

## Windows 环境注意事项

- 避免将项目放在包含权限限制或同步冲突的目录中
- 安装 Nmap 后确认命令行可直接调用
- 如 PowerShell 执行策略限制脚本运行，请按本机策略处理

## Linux 环境注意事项

- 确保 Java、Node.js、MySQL、Nmap、Nuclei 都已安装
- 如使用 Docker 启动，请确认当前用户具备 Docker 权限
- 部署时建议将数据库、日志与上传目录独立持久化
