# 部署说明

## 部署方式概览

项目支持两类常见部署方式：

- 通过 Docker Compose 进行一体化部署
- 通过手动构建前后端并运行后端 Jar 包进行部署

## Docker Compose 部署

```bash
docker-compose up -d --build
```

部署前建议：

- 复制 `.env.example` 为本地环境变量文件
- 修改数据库密码、JWT 密钥和 AI 配置
- 确认主机已安装 Docker 与 Docker Compose

## 手动部署

### 构建前端

```bash
cd frontend
npm install
npm run build
```

### 构建后端

```bash
cd backend
mvn clean package -DskipTests
```

### 运行后端

```bash
java -jar target/server-scout-1.0.0.jar
```

## 数据库配置

关键配置项：

- `SPRING_DATASOURCE_URL`
- `SPRING_DATASOURCE_USERNAME`
- `SPRING_DATASOURCE_PASSWORD`

请确保数据库已存在且已执行 `serverscout-init.sql` 初始化脚本。

## 环境变量说明

- `SERVER_PORT`：服务端口
- `JWT_SECRET`：JWT 密钥
- `NMAP_PATH`：Nmap 路径
- `NUCLEI_PATH`：Nuclei 路径
- `AI_BASE_URL`：AI 接口地址
- `AI_API_KEY`：AI 接口密钥
- `PDF_FONT_PATH`：PDF 中文字体路径

## Nmap / Nuclei 路径配置

- 若系统 PATH 中已存在命令，可直接使用默认值
- 若命令不在 PATH 中，请通过环境变量指定完整可执行路径

## 日志目录

- 后端运行日志默认写入应用日志目录
- Docker Compose 场景下，日志通过卷挂载到容器内 `/app/logs`

## 文件上传目录

当前项目应根据实际后端实现配置上传或截图目录，如未启用相关功能，可保持默认。

## 报告导出目录

PDF / Excel 报告由后端按请求即时生成，具体持久化方式以运行环境配置为准。

## 部署后检查

- 前端页面能否正常打开
- 后端 `/docs` 是否可访问
- 登录、资产列表、扫描任务是否可正常加载
- Nmap / Nuclei 调用路径是否正确
- 数据库连接和表结构是否正常

## 常见问题

- 容器启动失败：检查数据库密码、端口占用和环境变量
- 后端报数据库异常：确认连接地址、账号和初始化脚本
- 扫描无法执行：确认 Nmap、Nuclei 已安装且路径正确
- 报告导出乱码：确认字体文件已安装并正确配置 `PDF_FONT_PATH`
