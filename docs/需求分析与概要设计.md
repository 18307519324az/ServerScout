### 1. 项目背景

#### 1.1 行业背景

随着企业数字化转型深入，IT 资产规模快速增长，暴露的攻击面也日益扩大。据 2025-2026 年安全行业报告：

- 超过 60% 的企业无法准确掌握自身互联网资产全貌
- 平均每个企业存在 200+ 无人维护的"影子资产"
- 新漏洞从披露到被利用的平均时间已缩短至 15 天

攻击面管理（Attack Surface Management, ASM）已成为 2025-2026 年网络安全领域最热门的方向之一。Gartner 将其列为顶级安全技术趋势。

#### 1.2 项目定位

ServerScout 是一个**轻量级的服务器资产攻击面可视化分析平台**，面向中小型企业安全运维人员，提供从资产发现、指纹识别到漏洞关联的全流程攻击面管理能力。

有别于 Goby、Nessus 等商业重量级扫描器，ServerScout 的定位是：

- **轻量易部署**——基于 Spring Boot，开箱即用
- **可视化优先**——以拓扑图、仪表盘等直观方式展示攻击面
- **可扩展**——插件式扫描引擎，支持集成 Nmap、Nuclei 等开源工具
- **聚焦 Web 资产**——重点关注 Web 服务、API 端口等常见攻击入口

#### 1.3 目标用户

| 用户角色 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| 安全运维人员 | 日常资产巡检、漏洞排查 | 快速发现新增资产和潜在风险 |
| 渗透测试人员 | 信息收集阶段 | 快速了解目标资产全貌和攻击面 |
| 系统管理员 | 服务器管理 | 掌握服务器开放端口和服务版本 |

### 2. 用户需求

#### 2.1 用户痛点

1. **资产不清**：服务器数量多，IP 分散，难以形成统一资产视图
2. **扫描靠人工**：每次安全检查需要手动执行 Nmap 命令，结果零散难以管理
3. **漏洞响应滞后**：不知道哪些资产受新公开漏洞影响，修复不及时
4. **报告靠手写**：巡检报告需要人工整理，效率低且格式不统一
5. **缺乏可视化**：纯文本的扫描结果难以快速理解攻击面全貌

#### 6.1 扫描引擎模块（策略模式）

```java
// 策略接口
public interface ScannerStrategy {
    ScanResult execute(ScanTask task);
    ScannerType getType();
}

// Nmap 实现
@Component
public class NmapScannerImpl implements ScannerStrategy {
    public ScanResult execute(ScanTask task) {
        // 1. 构建 Nmap 命令
        // 2. ProcessBuilder 执行
        // 3. 解析 XML 输出
        // 4. 返回 ScanResult
    }
}

// Nuclei 实现
@Component
public class NucleiScannerImpl implements ScannerStrategy {
    public ScanResult execute(ScanTask task) {
        // 1. 构建 Nuclei 命令
        // 2. 执行扫描
        // 3. 解析结果
    }
}
```

**扫描流程**：
```
创建任务 → 异步执行器启动
    → 调用 Nmap CLI（端口扫描）
    → 解析 XML 结果 → 资产+端口入库
    → 更新任务进度
    → 调用指纹识别
    → 调用 CVE 关联
    → 任务完成
```

#### 6.2 可视化模块

**仪表盘**：
- 总资产数、总端口数、总漏洞数（统计卡片）
- 高危漏洞数（醒目红色标识）
- 端口分布 Top 10（柱状图）
- 漏洞等级分布（饼图）
- 资产发现趋势（折线图）

**拓扑图**（D3.js 力导向图）：
- 节点：每个资产为一个节点
- 颜色：按高危漏洞数分级（绿→黄→橙→红）
- 交互：悬停摘要、点击跳转详情
