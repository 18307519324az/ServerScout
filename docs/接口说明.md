# 接口说明

## 接口文档入口

项目后端已集成 Swagger / OpenAPI。

- Swagger UI：`http://localhost:8080/docs`
- OpenAPI JSON：`http://localhost:8080/api-docs`

## 认证方式

项目采用 Spring Security + JWT 认证方式。

- 登录成功后由后端返回 Token
- 前端在后续请求中携带 Token 访问受保护接口
- 权限区分通过角色信息完成

## JWT 使用说明

- `JWT_SECRET` 通过环境变量配置
- Token 过期时间由 `JWT_EXPIRATION_MS` 控制
- 前端应在本地存储 Token，并在过期或失效后重新登录

## 统一响应结构

项目存在统一响应封装，典型结构如下：

```json
{
  "code": 20000,
  "message": "success",
  "data": {}
}
```

## 统一错误码说明

- `20000`：成功
- `400xx`：请求参数或校验错误
- `401xx`：未认证或 Token 失效
- `403xx`：权限不足
- `404xx`：资源不存在
- `409xx`：数据冲突
- `500xx`：服务端内部错误或扫描、第三方服务异常

## 典型接口说明

### 认证登录

- 作用：完成用户登录并返回认证结果
- 模块：认证模块

### 扫描任务创建

- 作用：提交目标范围和扫描参数，创建扫描任务
- 模块：扫描任务模块

### 任务进度订阅

- 作用：通过 SSE 监听扫描任务进度
- 模块：扫描任务模块

### 资产查询

- 作用：获取资产列表和资产详情
- 模块：资产模块

### 报告导出

- 作用：导出 PDF 或 Excel 报告
- 模块：报告模块

### AI 风险简报

- 作用：根据输入证据生成结构化风险摘要
- 模块：AI 模块

## 接口调试说明

- 建议优先以 Swagger / OpenAPI 页面为准
- 若本地未启用接口文档，请根据控制器代码和请求结构进行调试
- 涉及鉴权接口时，请先登录获取 Token 再发起请求
